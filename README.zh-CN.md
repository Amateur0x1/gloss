# Gloss

[English README](./README.md)

Gloss 是一个本地优先的 macOS 桌面应用，用于多语言对读、段落对齐与细读研究。

当前版本更偏向学术使用场景：导入两份文档后，应用会在本机完成文本提取、句段切分、向量生成与跨语言检索。你点击一侧的句段时，另一侧会返回最可能对应的原文片段。

## 为什么叫 Gloss

`Gloss` 在文本学、语言学里有“注释、训释、旁注”的意思，也很适合这个项目的方向：

- 支持译本与原文的对照阅读
- 帮助定位可能对应的源文本段落
- 适合做带研究性质的细读
- 不把产品限制在 PDF 这一种格式上

## 当前功能

- 基于 Electron、React、TypeScript 的本地 macOS 桌面应用
- 导入两份文档做并排对照
- 本地解析 PDF 文本
- 将文本切成句子级或短段落级片段
- 在本机生成多语言 embedding
- 点击中文句段时，自动检索最可能对应的英文片段
- 返回多条候选结果，而不是只返回一个结果
- 使用后台 Worker 处理重计算，避免界面主线程被卡死

## 工作方式

Gloss 当前的大致流程是：

1. 导入源文档和目标文档。
2. 从 PDF 中提取纯文本。
3. 重建行文本并切成较小的句段。
4. 使用多语言句向量模型编码这些片段。
5. 在跨语言向量空间中比较相似度。
6. 用轻量的邻域上下文做一次重排。
7. 在目标侧高亮并滚动到最优命中的片段。

## 模型

Gloss 当前使用的是 `@huggingface/transformers` 加载的多语言 embedding 模型：

- `Xenova/paraphrase-multilingual-MiniLM-L12-v2`

需要注意：

- 首次运行会下载模型资源到本地缓存
- 模型仓库里包含多个不同精度和量化版本，当前应用走的是量化后的、适合本地浏览器/桌面运行的 ONNX 路径
- 这是一条务实的第一版检索方案，不是最终形态的学术级精确对齐模型

## 项目结构

```text
electron/                Electron 主进程与 preload
src/App.tsx              主界面
src/lib/pdf.ts           本地 PDF 文本提取与句段切分
src/lib/embeddings.ts    模型加载与向量生成
src/lib/alignment.ts     跨语言相似度计算与轻量重排
src/workers/             后台文档处理 Worker
```

## 当前限制

这仍然是一个早期研究原型。

目前的限制包括：

- 现在的对齐是基于“提取后的文本片段”，不是基于 PDF 原始版面坐标
- 匹配效果很依赖 PDF 文本提取质量
- 如果译文存在大量意译、合并拆句、删改，命中准确率会下降
- 当前交互首先围绕“中文查英文”这个方向优化
- 还没有持久化的本地书库和向量索引缓存层

## 后续规划

下一步适合继续推进的方向有：

- 双向检索
- 更强的上下文重排
- 支持 PDF 以外的更多文档格式
- 持久化本地书库与索引
- 页面级 PDF 阅读与可视化高亮
- 更适合学术研究的批注和笔记工作流

## 目标用户

Gloss 当前尤其适合：

- 学者
- 译者
- 需要做双语细读的学生
- 处理平行文本的研究者

## 当前状态

Gloss 已经可以作为一个本地多语言片段检索原型来使用，但还不是一个完成度很高的正式生产级阅读环境。
